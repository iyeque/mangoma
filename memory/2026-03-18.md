## Mission: Refactor Live Streaming Backend for Continuous Lyria Generation

### Research Complete: Lyria API Insights

**Source:** `@google/genai` TypeScript definitions from node_modules

**Key Discovery:**
Lyria's `lyria-realtime-exp` model supports TWO modes:
1. **Audio input (current)**: `sendRealtimeInput({ audio })` - requires microphone
2. **Text prompt mode (desired)**: `setWeightedPrompts()` + `setMusicGenerationConfig()` - autonomous generation

**Continuous Generation Flow:**
```typescript
const session = await ai.live.connect({ model: 'models/lyria-realtime-exp' });
await session.setWeightedPrompts({
  weightedPrompts: [{ text: "Lo-fi chill beats, 70 BPM", weight: 1.0 }]
});
await session.setMusicGenerationConfig({
  musicGenerationConfig: { bpm: 70, temperature: 0.7, density: 0.6 }
});
session.play(); // Starts continuous streaming

// Listen for audio chunks
session.on('message', (msg) => {
  const audio = msg.audioData; // Forward to YouTube RTMP
});

// Mid-stream updates:
await session.setMusicGenerationConfig({ bpm: 85 });
await session.setWeightedPrompts({ text: "Energetic synthwave" });
```

**Critical Parameters:**
- `bpm`: 60-200
- `temperature`: 0.0-3.0 (variance)
- `guidance`: 0.0-6.0 (prompt adherence)
- `density`: 0.0-1.0 (sound density)
- `brightness`: 0.0-1.0
- `muteBass`, `muteDrums`, `onlyBassAndDrums`: boolean filters

**Mood/Genre Control:** Conveyed through the **text prompt**, not separate enum fields. Example: `"chill lo-fi hip-hop"` vs `"energetic synthwave retro"`.
