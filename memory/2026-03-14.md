1. **Audio Format**: Keep WebM/Opus from MediaRecorder → Lyria, transcode to AAC only at RTMP output (FFmpeg)
   - Rationale: Opus is superior quality, single conversion point, low latency

   - Will extend/reuse `@google/genai` SDK

## Implementation Phase (19:00-19:30)

### Created Live Streaming Module Structure
```
mangoma/live/
├── package.json           # Dependencies (TypeScript, ws, express, @google/genai)
├── tsconfig.json          # TypeScript config
├── .env.example           # Environment template
├── src/
│   └── server.ts          # Main WebSocket + HTTP server (1075 lines)
└── presets/               # Directory for JSON preset files
```

### Server Features Implemented
- **WebSocket endpoint**: `/live/stream` (accepts binary audio + JSON control messages)
- **HTTP API**:
  - `GET /health` — health check
  - `GET/POST/DELETE /api/presets` — preset CRUD
  - `POST /api/stream/start` — begin streaming with preset
  - `POST /api/stream/stop` — stop streaming
  - `POST /api/stream/update` — real-time parameter changes
  - `GET /api/stream/status` — current state
- **Orchestration**:
  - LyriaClient (to be implemented) for Google Lyria API
  - YouTubeRTMPStreamer for FFmpeg RTMP output
  - PresetsAPI for JSON preset management
- **Control handlers**:
  - `preset` message type — load preset
  - `update` message type — adjust BPM/mood/genre/intensity in real-time
  - `ping/pong` — keepalive
- **Graceful shutdown** — SIGTERM/SIGINT cleanup

### Next Files to Create
- `src/lyria_client.ts` — Google Lyria WebSocket integration
- `src/youtube_rtmp.ts` — FFmpeg child process for RTMP streaming
- `src/presets_api.ts` — Filesystem-based preset store
- `src/config.ts` — Environment config loader
- `src/utils.ts` — Logging and helpers
- `live/frontend/index.html` — Test page with MediaRecorder

### Design Decisions
- Use `ws` library for WebSocket server (lightweight, no framework overhead)
- Preset schema includes:
  * `lyria.params` — { bpm, mood, genre, intensity, temperature? }
  * `youtube` — { streamKey, title, description, tags, chatModeration }
  * `interaction` — { chatCommands, realtimeAdjustments.cooldown }
- Single-stream concurrency model (one active stream per server instance)
- JSON control messages over same WebSocket as binary audio
